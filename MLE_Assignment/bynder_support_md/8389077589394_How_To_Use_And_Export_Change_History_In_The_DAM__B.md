# How To Use And Export Change History In The DAM – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/8389077589394-How-To-Use-And-Export-Change-History-In-The-DAM

Summary[](https://support.bynder.com/hc/en-us/articles/8389077589394-How-To-Use-And-Export-Change-History-In-The-DAM#h_01JXVRVD8MESXCJXX4TQVX7E78)

---------------------------------------------------------------------------------------------------------------------------------------------------

Bynder users can monitor modifications made on your portal in the **Change History** section. Here, you can view updates to Users, Permissions, Assets, Taxonomy, Taxonomy Configuration, Collections, and Workflow.

Who?[](https://support.bynder.com/hc/en-us/articles/8389077589394-How-To-Use-And-Export-Change-History-In-The-DAM#h_01JXVRXSFJ608GY370W8NJE6KF)

------------------------------------------------------------------------------------------------------------------------------------------------

This feature/solution requires your Customer Success Contact to enable, but then individual permissions can be done by the Bynder Admin.

Once the feature is enabled users with the _View Change History_ permission can see Change History. Users will need to also have the _Export User Data_ permission to export the change history into a CSV.

Why?[](https://support.bynder.com/hc/en-us/articles/8389077589394-How-To-Use-And-Export-Change-History-In-The-DAM#h_01JXVS4R7QYNSR6ECTZPHA4JEV)

------------------------------------------------------------------------------------------------------------------------------------------------

Some changes need to be monitored, or even undone. Users have the ability to track the changes made to their DAM to ensure mistakes can be corrected, and assets are secure. Using the change history option is an easy to use added safety and management feature of the Bynder DAM.

How?[](https://support.bynder.com/hc/en-us/articles/8389077589394-How-To-Use-And-Export-Change-History-In-The-DAM#h_01HSGXA3R7MF3YPTKREKR0ANWY)

------------------------------------------------------------------------------------------------------------------------------------------------

### Viewing The Change History

1.  Navigate to **Settings** > **Reporting** > **Change** **history**.**![](https://support.bynder.com/hc/article_attachments/16717344000658)**
2.  On the lefthand sidebar, select the section you want to explore: **Users**, **Permissions**, **Assets**, **Metaproperties**, **Collections**, or **Workflow**. You can view sections by various filters within each section.

![screenshot_2025-01-07_at_15.59.50.png](https://support.bynder.com/hc/article_attachments/23768952899602)

### Changed Users

See changes made to user fields, the user's method to log in, or when a user is added or removed from a group. If a user is deleted, you can only see their User ID; their contact information will be removed from the portal. 

### Changed Permissions

See if permissions have been changed.

### Changed Assets

User with required permissions can see changes applied to assets such value changes to metaproperties, advanced rights, embargos, or deletion. Changes made with the bulk editor are not available in the change history if the individual assets are not available in the edit screen. When the bulk editor is triggered depends on configuration of the portal, the default value is 20. In the exported CSV all changes will be included also for the assets that got edited as a batch.

### Changed Taxonomy

See changes made to metaproperty, event types (i.e., created, deleted, imported), and the updated fields.

### Changed Taxonomy Configuration

Please see the changes made to the taxonomy configuration related to sorting (index) and whether the required status of a metaproperty has changed.

### Change Automations

See changes made to automations related action types, conditions, and triggers.

### Changed Collections

See changes made to collections created and shared.

### Changed Workflow Groups

See changes made to workflows in Workflow.  
  

### Exporting Change History Into A CSV

1.  **Navigate** to the Change History you would like to export. 
2.  **Click** the ![Screenshot 2024-01-30 at 2.22.22 PM.png](https://support.bynder.com/hc/article_attachments/16717344009362) button.
3.  Once the export is ready, a number will appear next to the one in the top righthand corner.   
4.  Click this, the Notification Center will open, and users can download the CSV file. ![Screenshot 2024-01-30 at 2.20.30 PM.png](https://support.bynder.com/hc/article_attachments/16717344019218)
