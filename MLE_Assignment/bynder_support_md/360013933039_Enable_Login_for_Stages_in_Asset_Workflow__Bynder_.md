# Enable Login for Stages in Asset Workflow – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/360013933039-Enable-Login-for-Stages-in-Asset-Workflow

If a workflow stage requires approval, and you want to make sure the right person approves it, you can set additional authentication requirement for the approval stage. In this way, the user who is responsible for the approval stage needs to log in again (reauthenticate) before they can submit the job to the next stage.

![authenticate-in-stage.png](https://support.bynder.com/hc/article_attachments/10108178123154)

### Note

*   Users who have **Allows users to get all the permissions available in the workflow** in their permission profile can overrule reauthentication.
    
*   The reauthentication requirement works in combination with the **Editable by** field. This means, that the user or users with the edit rights for the stage will be the ones required to log in again to submit the stage to the next level.
    

1.  Go to **Settings > Workflow > Preset Management**.
    
2.  Click the **Job** hyperlink.
    
3.  Select the preset you want to update.
    
4.  In the **Options** section for the approval stage, select Reauthenticate necessary before completing stage.
    
5.  Save your changes.
    

Results[](https://support.bynder.com/hc/en-us/articles/360013933039-Enable-Login-for-Stages-in-Asset-Workflow#h_01HP4TTZYM39FMZYC74DWPA5R2)

--------------------------------------------------------------------------------------------------------------------------------------------

Now, when users responsible for the job want to submit the job to the next stage, they are prompted to log in again. If the login is successful, users can click the button. If they do not click the button within 5 minutes, they have to reauthenticate again.

Learn more[](https://support.bynder.com/hc/en-us/articles/360013933039-Enable-Login-for-Stages-in-Asset-Workflow#h_01HP4TTZYMN3SER1XVEMGM5R89)
