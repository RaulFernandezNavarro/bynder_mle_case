# Working with Job Attachments in Asset Workflow – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/21838887479186-Working-with-Job-Attachments-in-Asset-Workflow

Job attachments in [Asset Workflow](https://support.bynder.com/hc/en-us/articles/360013932859)
 are files that can be added to a [job](https://support.bynder.com/hc/en-us/articles/360015349960)
 and made available at all workflow stages. These files typically provide important context for the content request, such as guidelines, reference materials, or briefs, and are separate from files submitted for review or approval. Any file type [supported by Bynder](https://support.bynder.com/hc/en-us/articles/360013936359)
 can be added as an attachment.

Job attachments are available in all presets that do not have the **Hide** **Attachment** option enabled.

You can add attachments directly to a [preset](https://support.bynder.com/hc/en-us/articles/18951075931666)
. This lets you automatically include specific files in every job created using that preset.

### Note

The number of attachments is not limited; however, the best practice is to keep it within 100 files.

1.  Open the job where attachments are supported.
2.  In the left sidebar, you can find the **Job** **Attachments** section.
3.  Drag and drop files directly into this section or click the **upload** area to select files manually.
4.  Select the ![](https://support.bynder.com/hc/article_attachments/21886739512722)next to the files you want to add as attachments.
    
    ![](https://support.bynder.com/hc/article_attachments/21886715746578)
    
5.  Select **Attach.**
6.  Once uploaded, these attachments will be accessible throughout the job’s workflow stages.
