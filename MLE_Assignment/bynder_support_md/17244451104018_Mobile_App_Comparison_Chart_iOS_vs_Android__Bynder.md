# Mobile App Comparison Chart (iOS vs. Android) – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/17244451104018-Mobile-App-Comparison-Chart-iOS-vs-Android

We offer the Bynder Mobile App for [iOS](https://support.bynder.com/hc/en-us/articles/360013871260)
 and [Android](https://support.bynder.com/hc/en-us/articles/17244072600594)
 Apps. See below for a comparison chart that outlines the difference in functionality for each device type. 

Mobile App Comparison Chart (iOS vs. Android)[](https://support.bynder.com/hc/en-us/articles/17244451104018-Mobile-App-Comparison-Chart-iOS-vs-Android#h_01HQBERRJXMY27JVCCSQN6HQXR)

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

|     |     |     |     |     |
| --- | --- | --- | --- | --- |
| **Categories** | **Working Functionality** | **Limitation** | **iOS App** | **Android App** |
|     |     |     |     |     |
| **Searching / Filtering** | Users are able to search by keyword tags | Users cannot search for AI tags | ✅   | ✅   |
|     |     |     |     |     |
|     | Users can view recent searches in the app | There currently is no way to filter the waiting room for app uploads |     |     |
|     |     |     |     |     |
|     | Users can filter using both advanced filters and smart filters |     | ✅   | ✅   |
|     |     |     |     |     |

|     |     |     |     |     |
|     |     | There is no ability to add metadata during mobile uploads |     |     |
|     |     |     |     |     |
|     |     | Required metaproperties are not respected |     |     |
|     |     |     |     |     |
| **Metadata** | Standard metadata: “Added on”, “Date modified”, “Publication date”, “Brand” | AI tags are not displayed in the app, but are displayed in the interface |     |     |
|     |     |     |     |     |
|     |     |     |     |     |
| **Downloading** | Disclaimers are respected | No derivatives in the mobile app, users always download the original file |     |     |
|     |     |     |     |     |
|     | Permissions are respected | PDF files can be viewed in the iOS native PDF viewer | ✅   | ✅   |
|     |     |     |     |     |
| **Sharing & Collections** | User can create new collections in the mobile app | Users can share assets, however they need to be downloaded first | ✅   | ✅   |
|     |     |     |     |     |
|     | Users can add assets to existing collections in the mobile app |     | ✅   | ✅   |
|     |     |     |     |     |
|     | The favorite assets are automatically sent to a “favorites” collection | Users can see all collections in the DAM, however they are blurred and cannot be added to | ✅   | ✅   |
|     |     |     |     |     |
|     |     | Users who log into the app with full collection permissions, have a holistic view of all collections in the DAM, which is different than the interface view | ✅   | ✅   |

| **Permissions** | Content in the app is respected by the user’s permissions |     | ✅   | ✅   |
|     |     |     |     |     |
| **Available** **Offline** | Only implemented in iOS | This feature is not available for Android | ✅   |     |
|     |     |     |     |     |

|     | You can see it in the interface as well, but it needs to be triggered in the app |     | ✅   |     |
|     |     |     |     |     |

|     |     |     |     |     |
|     | Android notification in the panel when upload is finished |     |     |     |
