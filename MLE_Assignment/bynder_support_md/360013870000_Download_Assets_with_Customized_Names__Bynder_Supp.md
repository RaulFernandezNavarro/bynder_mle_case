# Download Assets with Customized Names – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/360013870000-Download-Assets-with-Customized-Names

By default, when you download an asset from the Bynder Asset Library, the file is named according to the original filename used during upload. However, if you prefer to download files using the updated asset title or a customized naming convention, this can be enabled as an additional feature.

Once enabled, this feature can provide more user-friendly file names based on your preferred structure.

This feature/solution requires your Customer Success Contact to enable, but then individual permissions can be done by the Bynder Admin.

How to Set Up a Custom Naming Convention[](https://support.bynder.com/hc/en-us/articles/360013870000-Download-Assets-with-Customized-Names#h_01JC49S2VAN8JGPCSNRY10YYNN)

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------

When downloading assets, you can configure a custom naming convention by using metaproperties to define the file name. This makes file names more informative and aligned with your organizational needs.

For example, you might decide on a format like: **year-country-state-city-company-filetitle.jpg**

For instance, a file could be named: **2024-USA-Massachusetts-Boston-Bynder-example123.jpg**

Alternatively, you can exclude the asset title from the download name and keep other details like: year-country-state-city-company.jpg

If specific metaproperties aren’t filled in for an asset, the system will default to using the original filename upon download.

Important Notes[](https://support.bynder.com/hc/en-us/articles/360013870000-Download-Assets-with-Customized-Names#h_01JC49TD6FVHTAPH8FD9MFV63Q)

------------------------------------------------------------------------------------------------------------------------------------------------

*   Not all assets may have a title. If an asset lacks a title, the original filename will be used by default.
*   The asset title cannot include a question mark (?), as it is a reserved character in the HTTP protocol and may prevent successful downloading.
*   The capitalization of the download name will match the labels of your metaproperty options.
    *   For example, both 2022-France-Paris-Bynder.jpg and 2022-france-paris-bynder.jpg are possible, depending on your settings.
