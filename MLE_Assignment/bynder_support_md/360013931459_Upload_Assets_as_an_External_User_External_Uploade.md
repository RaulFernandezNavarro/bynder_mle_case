# Upload Assets as an External User (External Uploader) – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/360013931459-Upload-Assets-as-an-External-User-External-Uploader-

If you don't have a Bynder account and need to upload assets for approval, you can use the external uploader on the brand's Bynder login page.

How to Upload an Asset in External Uploader[](https://support.bynder.com/hc/en-us/articles/360013931459-Upload-Assets-as-an-External-User-External-Uploader-#h_01JBYZDH2VNCGNG8RE78FN2STE)

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.  Navigate to the brand portal URL.
    
2.  Drag and drop the assets into the **External Uploader** field ![Screenshot 2024-06-12 at 11.07.29 AM.png](https://support.bynder.com/hc/article_attachments/19503777687570)
    
    *   There is no limit to the number of files per upload batch. The default maximum file size per file is 50GB. [Unsupported browsers](https://support.bynder.com/hc/en-us/articles/360013873080)
         may have lower file size limitations.
3.  The upload will start automatically.
    
4.  Enter the email address of one or more recipients or select from the predefined list of recipients. Make sure to separate email addresses with a comma.![Screenshot 2024-06-12 at 10.59.49 AM.png](https://support.bynder.com/hc/article_attachments/19503777697938)
    
5.  Enter your email address.
    
6.  (Optional) Type a message to the recipient.
    
7.  (Optional) If required, provide the archive date.
    
8.  Click **Send files**.
    
9.  The files are uploaded to the portal's [Waiting Room](https://support.bynder.com/hc/en-us/articles/360013870280)
     for audit. The recipient and sender will receive an email notification that the files have been uploaded, and a direct link to download the files will be included. 
    

_Note: The asset will be uploaded with the **added by** section filled in as **external user.**_
