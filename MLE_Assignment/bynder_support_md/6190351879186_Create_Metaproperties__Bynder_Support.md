# Create Metaproperties – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/6190351879186

[Metaproperties](https://support.bynder.com/hc/en-us/articles/19460911311506)
 and their [options](https://support.bynder.com/hc/en-us/articles/10341416276370)
 are essential to your Bynder [taxonomy](https://support.bynder.com/hc/en-us/articles/17558350591762)
 and should be configured before you [upload your assets.](https://support.bynder.com/hc/en-us/articles/360013931479)

Once you've created your metaproperties, you can [configure,](https://support.bynder.com/hc/en-us/articles/6190359503634)
 [edit, or delete](https://support.bynder.com/hc/en-us/articles/19460749107218)
 the options in your portal under **metaproperties** **management**. 

**Who can manage metaproperties?**

Users with the following rights can manage metaproperties:

*   [**Manage metaproperties**](https://support.bynder.com/hc/en-us/articles/18433246503442-Permissions-in-Asset-Bank#:~:text=Manage%20metaproperties%20%2D%20Allows%20users%20to%20manage%20metaproperties.)
    

Create Metaproperties[](https://support.bynder.com/hc/en-us/articles/6190351879186#h_01J01JW55SF4KKT67DSS7XV2JE)

-----------------------------------------------------------------------------------------------------------------

1.  Navigate to ![settings_menu.png](https://support.bynder.com/hc/article_attachments/10433841600402) **Settings** > **Taxonomy** > **Metaproperties** **management** and click **Metaproperties** **and** **options** in the left sidebar.
    
    ![METAPROPERTY_AND_OPTIONS_-_NEW_TAXONOMY_.png](https://support.bynder.com/hc/article_attachments/10433872281746)
    
2.  Click **New metaproperty** in the top right corner.
    
    ![as_new_metaproperty_new_taxonomy.png](https://support.bynder.com/hc/article_attachments/10433872321426)
    
3.  In the **Name** field, enter a unique name for the new metaproperty. The Database name is automatically created based on the **Metaproperty** **Name** (Label). 
    
    ![step_one__new_metaproperty.png](https://support.bynder.com/hc/article_attachments/10433841746450)
    
    *   Note: The "unique name" cannot be changed; however, you can change the label if you decide to change the name of the selected metaproperty.
4.  Select the **Type** for the metaproperty. The **Type** **Preview** section of the screen shows a preview of the type and the user experience.
    
    ![metaproperty_type.png](https://support.bynder.com/hc/article_attachments/10433872411794)
    
    ### Note
    
    Once you assign a type to your metaproperty, you can only alternate the **Multi-select** and **Single** **select** options. No other type can be changed after saving. If you switch the metaproperty from **Multi** **Select** to **Single** **Select**, assets will remain tagged with multiple options.
    
    ![single_select_metaproperty_new_.png](https://support.bynder.com/hc/article_attachments/10433872442642)
    
    *   **Single select**
        
        Allows users to select one option from a list of metaproperty options. For example, a list of asset types where only one should be selected, such as image, document, video, etc.
        
    *   **Multi-select**
        
        It allows users to select multiple options from a list of metaproperty options, such as a list of multiple regions where an asset may be used, such as the Americas, EMEA, or APAC.
        
    *   **Text**
        
        Allows users to enter free-form text, such as the restrictions of an asset. 
        
    
    _Note: There is a 2500 character limit._
    
    *   **Date**
        
        Displays a date picker to apply a month, day, and year to an asset, such as a deadline. Using a date picker avoids human error and confusion around different ways of date formatting.
        
    
5.  If you’re creating a **Single** **Select** or **Multi-select** metaproperty, you will see the following settings:
    
    ![new_metaproperty_multi_select.png](https://support.bynder.com/hc/article_attachments/10433872462866)
    
    *   **Autocomplete search:** Enable the metaproperty options to display as a dropdown with search functionality instead of a list with options in the upload screen. If enabled, users can start typing the option they're looking for or scroll through the list of options. This is helpful if you have a long list of options.
        
    *   [**Add new options:**](https://support.bynder.com/hc/en-us/articles/360013869840)
         Enable users to create new metaproperty options on the fly when uploading and editing new assets. If disabled, users can only choose from metaproperty options added by the admin.
        
    *   **Options:** Enter the metaproperty options you want to add to the metaproperty (these can also be added later).
        
        *   Press **Enter** to add the option and click ![trash.png](https://support.bynder.com/hc/article_attachments/10433872492562) to delete an option.
            
        *   Click ![configure-users_plus-sign.png](https://support.bynder.com/hc/article_attachments/10433841987858) the **Add** **option** to add another option
            
        
    
6.  Click **Create** **metaproperty.**
    

Make Metaproperties Required[](https://support.bynder.com/hc/en-us/articles/6190351879186#h_01HEWSVCQXWXSSQCFDAMQVKDTH)

------------------------------------------------------------------------------------------------------------------------

1.  Navigate to your Portal > ![settings_menu.png](https://support.bynder.com/hc/article_attachments/27567195314066) **Settings > Taxonomy > Metaproperties management** and click **Configuration** in the left sidebar.
    
2.  Go to the metaproperty you want to make required and click the **Required**![toggle_android.png](https://support.bynder.com/hc/article_attachments/27567195315090). The switch will no longer be grayed out.
    
3.  Users must now fill in this metaproperty when uploading and editing assets.![required_.png](https://support.bynder.com/hc/article_attachments/27567201592466)
    

Make Metaproperties Optional[](https://support.bynder.com/hc/en-us/articles/6190351879186#01J01H6MN05WRNQ26BPQQ9CVZD)

----------------------------------------------------------------------------------------------------------------------

By default, metaproperties are optional. If you set a metaproperty to required before, follow the steps below to make it no longer required.

1.  Navigate to your Portal > ![settings_menu.png](https://support.bynder.com/hc/article_attachments/27567195314066)**Settings** >**Taxonomy** > **Metaproperties** **management** and click Configuration in the left sidebar.
2.  Go to the metaproperty you no longer want to be required and click the **Required** switch. The switch will become greyed out.
    

Guidelines for Metaproperty Database Name[](https://support.bynder.com/hc/en-us/articles/6190351879186#h_01JD095DQ53X9XDP7Y7YTZC178)

-------------------------------------------------------------------------------------------------------------------------------------

When you create a [metaproperty](https://support.bynder.com/hc/en-us/articles/6190351879186)
, or [metaproperty option](https://support.bynder.com/hc/en-us/articles/10341416276370)
, a database name is automatically generated from the name.

*   Use only alphanumeric characters (A-Z, a-z, 0-9), hyphens (-), underscores (\_).
*   Any unsupported character will automatically be replaced with an underscore (\_).
*   Hyphens (-) cannot be the final character in a database name; if they are, they will be converted to an underscore (\_).
*   The database name must be unique. If a duplicate database name is found (e.g., from another metaproperty or option), a numeric suffix (\_1, \_2, etc.) is added to ensure uniqueness.
*   The database name is fixed at the time of creation. Future updates to the name in the taxonomy will not affect the database name.

Recommendation for Integrations[](https://support.bynder.com/hc/en-us/articles/6190351879186#h_01JD096NC6TQG0C53B37FYW3WD)

---------------------------------------------------------------------------------------------------------------------------

*   Avoid relying on the database name for integrations due to its restrictions and immutability.
*   Use the [externalReference](https://bynder.docs.apiary.io/#introduction/changelog/2024:~:text=to%20set%20an-,externalReference,-on%20options%20in)
     field instead. This field supports customer-specific logic, has no character restrictions, and can be updated to align with integration requirements.

**Where can I find the database name?**

The database name is visible only to those who have [managed metaproperty permission](https://support.bynder.com/hc/en-us/articles/18433246503442-Permissions-in-Asset-Bank#:~:text=Manage%20metaproperties%20%2D%20Allows%20users%20to%20manage%20metaproperties.)
. It appears in the metaproperty details under the metaproperty management section.

**Can I change the database name?**

No, after creating your metaproperty or metaproperty option, the database name becomes permanent and cannot be altered. However, you can change the name.

![Screenshot 2024-11-18 at 2.52.14 PM.png](https://support.bynder.com/hc/article_attachments/27566994160786)

For example, if you created the meta property **"**city**"** but wanted to change it to display "the city", you can edit the name; however, the database name will not reflect the updated name. 

**Can I add a special characters to a name?**

Yes, special characters are allowed in the name.
