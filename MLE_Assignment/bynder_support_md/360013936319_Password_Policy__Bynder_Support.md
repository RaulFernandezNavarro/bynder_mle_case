# Password Policy – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/360013936319-Password-Policy

A strong password policy is essential for maintaining portal security. Users should be required to create strong passwords, regularly change them, and prevent reusing previous passwords. The following are effective steps to prevent weak password practices.

Single Sign-On (SSO)[](https://support.bynder.com/hc/en-us/articles/360013936319-Password-Policy#h_01J0M8P77W85E46C89DR0TF821)

-------------------------------------------------------------------------------------------------------------------------------

[Enable (SSO) single sign-on](https://support.bynder.com/hc/en-us/articles/360013936419)
 in Bynder to eliminate users needing to remember a new password for their Bynder account.

Password Requirements[](https://support.bynder.com/hc/en-us/articles/360013936319-Password-Policy#h_01J0M8PF7DEHF81N94Z0A03WBK)

--------------------------------------------------------------------------------------------------------------------------------

Bynder passwords must be at least six characters, which can be increased (e.g., 14 characters). Additional criteria include lowercase and uppercase letters, numbers, and special characters (e.g., @, #, ! %, $). Customize criteria as needed. Contact your Customer Success Contact for specific requirements.

Enable Strong Passwords[](https://support.bynder.com/hc/en-us/articles/360013936319-Password-Policy#h_01J0M8PQ1QDXWTH1F6PYSYHZ3H)

----------------------------------------------------------------------------------------------------------------------------------

To enforce strong passwords, set requirements such as:

*   Minimum eight characters
*   No all-numeric or all-alphabetical groups
*   No consecutive identical characters
*   Combination of uppercase, lowercase, numeric, and special characters
*   It cannot contain parts of the username or previous passwords

Contact your Customer Success Contact to customize a message that will appear if users create weak passwords.

Prevent Reusing Passwords[](https://support.bynder.com/hc/en-us/articles/360013936319-Password-Policy#h_01J0M8RQEN60T62VFJ0GEM8HYY)

------------------------------------------------------------------------------------------------------------------------------------

Enable the password-policy user setting to prevent users from reusing previous passwords. The system checks a set number of previous passwords for uniqueness.

Mandatory Password Reset[](https://support.bynder.com/hc/en-us/articles/360013936319-Password-Policy#h_01J0M8S1DCGTNVC0DATRW8CT3Y)

-----------------------------------------------------------------------------------------------------------------------------------

Regular password updates can be required. This feature targets users with a password for their Bynder account (excluding SSO-only users). Users will need to reset their password based on:

### User Creation Date

Users who have never reset their passwords will use the account creation date as the reference. If the specified time frame has elapsed, they will be prompted to reset their password.

### Previous Password Reset

The last reset date will be the reference for users who have reset their passwords. Once the specified period elapses, users will be prompted to reset their passwords.
