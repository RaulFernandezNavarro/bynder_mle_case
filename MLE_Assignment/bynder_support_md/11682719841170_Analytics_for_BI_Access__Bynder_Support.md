# Analytics for BI Access – Bynder Support

**Source URL:** https://support.bynder.com/hc/en-us/articles/11682719841170-Analytics-for-BI-Access

Analytics for BI allows you to have a better understanding of how specific assets, collections, and users are performing. It provides a self-serve and secure connection to access data in real-time to create tailored dashboards and reports using your own business intelligence (BI) data analysis tools. Combine the Analytics BI with other Bynder APIs to create even more granular insights.

You may find the Analytics API documentation [here](https://bynder.docs.apiary.io/#reference/analytics)
 and the full API documentation [here](https://bynder.docs.apiary.io/)
.

How to Enable the Analytics for BI Access[](https://support.bynder.com/hc/en-us/articles/11682719841170-Analytics-for-BI-Access#h_01H9RDXYFVJS1M1V71RHSHM0VA)

--------------------------------------------------------------------------------------------------------------------------------------------------------------

Contact your Customer Success Contact to learn more about the Analytics for BI Access.

It takes up to 24 hours for the information to appear in the Analytics for BI Access.

Connect to the Analytics for BI Access[](https://support.bynder.com/hc/en-us/articles/11682719841170-Analytics-for-BI-Access#h_01H9RDXYFVYTHQZYF0CB1R8ZAC)

-----------------------------------------------------------------------------------------------------------------------------------------------------------

Follow these steps to connect to the Analytics for BI Access:

1.   You can retrieve the auth token for the Analytics for BI Access with the credentials of the OAuth App. Learn more about creating OAuth Apps [here](https://support.bynder.com/hc/en-us/articles/360013875180)
    .
2.  The portal admin should complete the following:
    *   Check the asset scope to retrieve data on assets. Learn more [here](https://bynder.docs.apiary.io/#reference/assets/asset-operations/retrieve-assets)
        .
    *   Check the admin user scope to retrieve data on users. Learn more [here](https://bynder.docs.apiary.io/#reference/users/user-operations/retrieve-users)
        . 
    *   Check the analytics read scope. Learn more [here](https://bynder.docs.apiary.io/#reference/analytics)
        . 
    *   Set the type of Grant for the Analytics for BI Access to **Client Credentials.  
        ![](https://support.bynder.com/hc/article_attachments/11682710177682)  
        **
3.  Use the token to access the Analytics for BI Access. Learn more [here](https://bynder.docs.apiary.io/#reference/oauth-2.0/token-endpoint/using-client-credentials)
    .

FAQ[](https://support.bynder.com/hc/en-us/articles/11682719841170-Analytics-for-BI-Access#h_01H9RDXYFV9EAQ4SSN3SRETJMZ)

### Once enabled, when will I see the information?

It may take up to 24 hours for information to appear.

### How much data can I access through Analytics for BI?

The API will give you access to the last 12 months of data.

### What kind of tokens do you need?

An Oauth2 token.

### How can I access the full history of data?

Contact your CSM to arrange a 1-time retrieval of all historical data.
