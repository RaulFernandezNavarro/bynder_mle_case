# How To Customize The Back Button In CX For User Community – Bynder Support

Summary[](https://support.bynder.com/hc/en-us/articles/22003429409938-How-To-Customize-The-Back-Button-In-CX-For-User-Community#h_01JNXFREAC80GADH1NEQVFSQNW)

--------------------------------------------------------------------------------------------------------------------------------------------------------------

In CX For User Community Landing Pages and Guides, you can customize the behavior of the back button that appears in the header of a Guide (nonpublic) or Landing Page. This feature gives you the option to redirect a user elsewhere.

Who?[](https://support.bynder.com/hc/en-us/articles/22003429409938-How-To-Customize-The-Back-Button-In-CX-For-User-Community#h_01JNXFV3XB1T6CW0QW45YYQ2VJ)

-----------------------------------------------------------------------------------------------------------------------------------------------------------

This feature/solution requires your Customer Success Contact to enable, but then individual permissions can be done by the Bynder Admin.

Once enabled users with the _Creator_ or _Manager_ permission create or customize a guide or landing page.

Why?[](https://support.bynder.com/hc/en-us/articles/22003429409938-How-To-Customize-The-Back-Button-In-CX-For-User-Community#h_01JNXFWYXQY9Y10VTB6CNMB7Y0)

-----------------------------------------------------------------------------------------------------------------------------------------------------------

Customizing the back button allows you to have even more control over the guides and landing pages in CXUC, and makes redirection possible. This easy and accessible customization is just one of the many ways Bynder empowers users to create the portal experience they desire.

How?[](https://support.bynder.com/hc/en-us/articles/22003429409938-How-To-Customize-The-Back-Button-In-CX-For-User-Community#h_01JFDNHMC22PNKTWVMMTDXPCBM)

-----------------------------------------------------------------------------------------------------------------------------------------------------------

1.  Navigate to your guide or landing page.
    
2.  Select **Edit**.
    
3.  Select **Theme.**
    
    ![Screenshot 2024-12-18 at 3.13.29 PM.png](https://support.bynder.com/hc/article_attachments/23390295320722)[](https://support.bynder.com/hc/en-us/articles/22003429409938-How-To-Customize-The-Back-Button-In-CX-For-User-Community#h_01JFDNMB866PTXNG977Y0C8B3J)
    
    -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
    
    *   You cannot adjust the back button behavior if you are using the default Bynder Theme.
4.  You will be directed to the **Properties** section**.**
5.   Select the **Navigation** dropdown.
6.  Scroll to the **Back Button Behavior** section.![Screenshot 2024-12-18 at 3.14.34 PM.png](https://support.bynder.com/hc/article_attachments/23390308075026)
    
7.  Toggle setting **on**.
8.  Select one of the following:
    
    *   **Browser back**: Leave as-is if you want the button to follow browser behavior.
    *   **Custom Link**: Enter a specific URL for redirection.**![Screenshot 2024-12-18 at 3.14.11 PM.png](https://support.bynder.com/hc/article_attachments/23390308075666)**
9.  **Save your changes** and test the behavior to ensure it aligns with the user experience goals.
    

### Redirecting to a Custom Section

You may prefer that users return to a certain Guide or Landing Page section. In this case, you can configure the custom link URL to end with directing the user to that section.

Use the **Custom Link** option to define the back button’s destination, bypassing the guide overview entirely.

**If users hit the browser’s back button, will they return to the overview page of the Landing Page or Guide?**

The browser back button follows standard browser behavior and takes the user to the previous page they visited. However, you can configure the **in-solution** back button that appears in the header of the page by redirecting users elsewhere using the "Custom Link" option.

**Can the guide overview page be completely hidden?**

While the guide overview cannot be removed entirely, it can be bypassed by directing users to specific pages using landing page buttons and configuring the back button with a custom link.

**What happens if the back button is set to "Browser Back"?**

It will take users to the previous page in their session history.
